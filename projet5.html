Projet - Automatisation de géotraitement QGIS avec script python et Model Builder

Exploration et visualisation des données touristiques collectées par web scraping

Contexte et objectifs

L’objectif de ce projet était de créer deux scripts de géotraitement afin de s’initier à l'utilisation de Python dans QGIS. Par la suite, j'ai pu les mettre en action via le modeleur graphique de QGIS.

Aperçu du Model Builder avec les scripts intégrés

Fonctionnement

À partir d’un fichier CSV listant les noms des tuiles raster associées à chaque entité spatiale étudiée, le premier script permet d’extraire automatiquement les tuiles correspondantes depuis un dossier contenant les données RGE Alti (téléchargées depuis le site Géoservices de l’IGN). Les tuiles sélectionnées sont ensuite sauvegardées dans un dossier spécifique. Le second script a pour but de fusionner ces tuiles en un seul fichier .TIFF. Grâce à la librairie GDAL, les tuiles sont regroupées et exportées vers un répertoire défini, avec possibilité d’affichage immédiat en tant que couche temporaire dans QGIS. Enfin, les deux scripts ont été intégrés dans le modeleur graphique de QGIS en tant que scripts de géotraitement. Cela permet d’automatiser l’ensemble du processus : détection des entités, génération du CSV, extraction des tuiles raster, puis leur fusion.
